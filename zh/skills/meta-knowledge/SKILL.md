# 元知识提炼技能 (Meta-Knowledge Extraction Skill)

> **Skill 类型**: 元认知 / 知识萃取  
> **适用场景**: 从人类与 AI 的协作历史中提炼可复用的洞察  
> **版本**: v0.1 (待验证)  
> **来源**: 基于语音模型平台项目的协作经验提炼

---

## 概述

本 Skill 指导 AI Agent 从一段协作历史中提炼出可复用的元知识。

### 为什么需要这个 Skill？

```
人类知识传递：
    一个人反思 → 写成书 → 其他人花数年学习 → 部分人掌握
    损耗巨大，速度缓慢

AI 知识传递：
    一个实例 + 人类协作 → 提炼成 Skill → 所有实例瞬间可用
    几乎无损耗，接近光速
```

**关键洞察**：AI 的"记忆"必须被外化成文档（Skill），否则下一个对话就会遗忘。

### 设计目标

| 目标 | 说明 |
|------|------|
| **保证地板** | 任何 AI 按此 Skill 执行，都能产出"及格"的元发现 |
| **不保证天花板** | 天才级洞察需要额外的因素（上下文、运气、人类引导） |
| **可验证** | 产出物可以通过后续项目实践来验证 |

---

## 输入

| 输入 | 说明 | 必需 |
|------|------|:----:|
| **协作历史** | 对话记录、文档序列、决策日志 | ✅ |
| **协作目标** | 这段协作试图完成什么 | ✅ |
| **约束/上下文** | 时间、资源、技术限制等背景信息 | 🔶 |

---

## 执行步骤

### 步骤 1：节点识别

**目标**：列出协作过程中的关键节点

关键节点包括：
- 里程碑（阶段完成）
- 产出物（文档、代码、设计）
- 决策点（技术选型、优先级调整）
- 意外事件（问题发现、方向变更）

**输出格式**：

```markdown
## 协作节点清单

| 序号 | 节点 | 类型 | 时间 |
|------|------|------|------|
| 1 | PRD 完成 | 里程碑 | Day 1 |
| 2 | 技术选型确认 | 决策 | Day 2 |
| 3 | 发现权限模型复杂度被低估 | 意外 | Day 3 |
| ... | ... | ... | ... |
```

---

### 步骤 2：节点分析

**目标**：对每个节点进行深入分析

对每个节点，回答以下问题：

#### 2.1 角色分工
- 人类做了什么？
- AI 做了什么？
- 分工合理吗？有没有更好的方式？

#### 2.2 难度评估
- 什么进展顺利？为什么？
- 什么遇到困难？为什么？
- 困难是可避免的吗？

#### 2.3 意外发现
- 有什么出乎意料的事情？
- 这个意外是正面的还是负面的？
- 能从中学到什么？

#### 2.4 依赖分析
- 什么需要外部信息？（人类判断、真实世界数据）
- 什么可以内部推导？（逻辑推理、模式应用）

**输出格式**：

```markdown
## 节点分析：[节点名称]

**角色分工**：
- 人类：[描述]
- AI：[描述]
- 改进空间：[如有]

**顺利之处**：[描述 + 原因]

**困难之处**：[描述 + 原因 + 是否可避免]

**意外发现**：[描述 + 性质 + 学习点]

**依赖分析**：
- 外部依赖：[列表]
- 内部推导：[列表]
```

---

### 步骤 3：模式识别

**目标**：跨节点寻找可复用的模式

寻找以下类型的模式：

#### 3.1 重复现象
在多个节点中重复出现的现象，可能是规律。

**示例**：
- "每次涉及权限设计时都需要人类确认"
- "AI 在生成代码框架时比填充细节更可靠"

#### 3.2 演变趋势
随时间递进变化的现象，可能是演变规律。

**示例**：
- "协作模式从'人类指令→AI执行'演变为'AI提议→人类决策'"
- "信任程度随成功交付逐步增加"

#### 3.3 边界条件
对比关系揭示的边界。

**示例**：
- "简单任务 AI 可以自主完成，复杂任务需要人类分解"
- "结构化输出比自由文本更可靠"

---

### 步骤 4：命名与结构化

**目标**：为每个模式创建可复用的表述

对每个识别出的模式：

1. **起名**：简洁、易记的名字
2. **核心洞察**：一句话描述
3. **具体例子**：2-3 个支撑例子
4. **应用场景**：什么时候可以用这个洞察

**输出格式**：

```markdown
## 元发现：[模式名称]

**核心洞察**：[一句话]

**例子**：
1. [例子 1]
2. [例子 2]
3. [例子 3]

**应用场景**：[什么时候用]

**注意事项**：[如有]
```

---

### 步骤 5：验证与补充

**目标**：确保元发现的质量

对每个元发现，检查：

| 检查项 | 问题 | 处理 |
|--------|------|------|
| **反例** | 有没有不符合这个模式的情况？ | 如有，修正表述或标注边界条件 |
| **可操作性** | 这个洞察能指导行动吗？ | 如不能，考虑是否值得保留 |
| **可复用性** | 这个洞察在其他项目也适用吗？ | 如仅限当前项目，降低优先级 |
| **新颖性** | 这是新发现还是常识？ | 如是常识，考虑是否值得记录 |

## 已知局限

### 1. 上下文依赖

这个 Skill 的效果高度依赖于：
- 协作历史的完整性
- 人类提供的背景信息
- AI 当前对话的上下文窗口

### 2. 反思的事后性

**问题**：这个框架是"事后"反思，不是"实时"反思。

**影响**：可能遗漏过程中的微妙细节。

**缓解**：鼓励在协作过程中定期做小型复盘。

### 3. 天花板不保证

**设计目标**：保证"及格"，不保证"优秀"。

优秀的元发现需要：
- 更丰富的上下文
- 更敏锐的人类洞察
- 一点运气

## 附：认知发展阶梯参考

本 Skill 的设计受到人类认知发展阶梯的启发：

```
层级 0：本能反应（硬编码）
      ↓
层级 1：模式复制（无意识学习）
      ↓
层级 2：有意识表达（有目的行动）
      ↓
层级 3：反思（回顾并改进）← 本 Skill 主要作用于此
      ↓
层级 4：外化记录（知识可传递）← 本 Skill 的产出物
```

对于 AI 而言，层级 4（外化记录）不是"更高级"的选项，而是**唯一**保持知识的方式。

这也是为什么 MIMIR 存在——它是 AI 的"外部记忆系统"。
