## Agent 提问脚本

当用户说"我要启动一个新项目"时，按以下顺序提问：

### 第一轮：项目基本分类

```markdown
好的，让我了解一下项目的基本情况：

1. **项目类型**：这是什么类型的软件？
   - [ ] Web 应用（浏览器访问）
   - [ ] 移动端 App
   - [ ] 桌面应用
   - [ ] CLI 工具 / 脚本
   - [ ] 后端服务 / API（无前端）
   - [ ] 其他：______

2. **项目性质**：
   - [ ] 企业级 / 商业项目（需要考虑安全、合规、扩展性）
   - [ ] 个人项目 / 实验性质
   - [ ] MVP / 快速验证

3. **简单描述**：用一两句话描述这个项目要做什么？
```

### 第二轮：根据类型深入（以企业级 Web 为例）

```markdown
了解了，这是一个企业级 Web 项目。让我再了解一些关键信息：

**用户与规模**：
1. 预计用户量级？（10人内 / 数十人 / 数百人 / 数千人 / 更多）
2. 并发用户数预估？
3. 是否需要支持多租户？

**认证与权限**：
4. 需要用户登录吗？
5. 有几种用户角色？权限模型复杂吗？
6. 未来是否需要对接企业 SSO（如 LDAP/SAML/OIDC）？

**数据与安全**：
7. 是否处理敏感数据（个人信息、金融数据等）？
8. 是否有合规要求（GDPR、等保等）？
9. 数据存储位置有限制吗（必须本地 / 可以上云）？

**技术约束**：
10. 公司现有技术栈是什么？有强制要求吗？
11. 团队熟悉哪些技术？
12. 有现成的基础设施（数据库、CI/CD 等）吗？

**时间与资源**：
13. 项目周期预期？
14. 开发团队规模？
```

## 分类结果映射

| 分类结果 | 加载的 Skill | 关键特点 |
|----------|--------------|----------|
| `enterprise-web` | `enterprise-web/SKILL.md` | 完整的安全、权限、审计、扩展性考量 |
| `lightweight-web` | `lightweight-web/SKILL.md` | 简化的认证，关注核心功能 |
| `mvp` | `mvp/SKILL.md` | 最小可行，快速迭代，技术债可接受 |
| `mobile-app` | `mobile-app/SKILL.md` | 原生/跨平台选型，App Store 发布 |
| `cli-tool` | `cli-tool/SKILL.md` | 参数解析，跨平台兼容 |
| `minimal` | 不加载特定 Skill | 直接开始，最小约束 |

---

## 信息收集完成后

1. **确认理解**：复述项目概况，确认理解正确
2. **加载 Skill**：根据分类加载对应的详细 Skill 文档
3. **开始执行**：按 Skill 指导的阶段逐步推进
4. **产出文档**：在关键节点产出对应的项目文档
