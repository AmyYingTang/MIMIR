# MIMIR

> **M**ethodology for **I**mplementation, **M**anagement & **I**ntelligent **R**eference

*像奥丁向智慧之泉的守护者请教一样，向 MIMIR 请教你的下一个项目。*

---

## 这是什么？

MIMIR 是一套**可执行的软件项目启动方法论**，专为 AI Agent（如 Claude）和人类开发者设计。

它不是又一本讲道理的方法论书籍，而是：

| 传统方法论 | MIMIR |
|-----------|-------|
| 静态知识，会过时 | 通过项目复盘持续更新 |
| 分散的概念 | 结构化的 Checklist + 决策树 |
| 读完还是不知道怎么做 | 直接产出项目文档 |
| 需要自己判断适用性 | 通过提问自动匹配场景 |

## 核心理念

```
真实项目经验 ──► 提炼方法论 ──► 指导新项目 ──► 复盘更新方法论
      ▲                                              │
      └──────────────────────────────────────────────┘
```

MIMIR 的名字来自北欧神话中**智慧之泉的守护者**。奥丁为了获取他的智慧，付出了一只眼睛的代价。这套方法论也是用真实项目的踩坑经验换来的。

## 快速开始

### 方式 1：配合 Claude 使用（推荐）

1. 在 [Claude.ai](https://claude.ai) 创建一个 Project
2. 将 MIMIR 的文件上传到 Project
3. 开始对话：

```
我要启动一个新项目，请用 MIMIR 帮我进行项目规划。

项目简介：[描述你的项目]
```

Claude 会根据 MIMIR 的指南，主动提问、给出建议、产出文档。

### 方式 2：人类直接使用

1. 从 `SKILL-INDEX.md` 开始
2. 根据你的项目类型，进入对应的 Skill 目录
3. 按照 Phase 顺序，逐步完成各阶段产出

## 亮点特性

### 🎯 决策树驱动

不是给你一堆知识让你自己选，而是通过提问帮你做决策：

```
Q: 预计用户量级？
   < 100      → 单体应用，单服务器
   100-1K     → 单体应用，可考虑读写分离
   1K-10K     → 负载均衡，缓存
   > 10K      → 微服务，容器编排
```

### 📋 Checklist 不遗漏

企业级项目容易遗漏的点都有提醒：

- 密码策略、登录保护、会话管理
- 敏感数据加密、日志脱敏
- 操作审计、数据保留
- 错误处理、超时控制、限流

### 🖥️ UI/UX 设计原则

基于真实项目经验提炼的前端交互指导：

- 终端用户使用向导式交互（Wizard Pattern）
- 管理员使用传统表格/表单布局
- 登录后引导页（Launchpad）替代 Dashboard
- 配置驱动的 UI 动态适配

### 🔄 复盘驱动更新

每个项目结束后，通过复盘萃取经验：

```
项目复盘 → 提炼可复用经验 → 更新 MIMIR → 下个项目受益
```

### 🤖 AI Agent 友好

专门为 AI Agent 设计的结构：
- 清晰的指令和产出要求
- 模板可以直接填充
- 检查清单可以逐项确认

## 示例：用 MIMIR 启动项目

### 输入

```
我要做一个企业内部的权限管理平台，
支持多级审批、角色权限分配、操作审计。
大概几百个用户，不需要高并发。
```

### MIMIR 会引导你完成

1. **需求分析** → 产出 PRD，明确用户角色、权限模型、P0-P3 功能划分
2. **技术选型** → 产出技术选型文档，确定技术栈
3. **系统设计** → 产出数据库设计、API 设计、UI/UX 设计原则
4. **详细规格** → 产出状态机定义、业务规则、UI 原型
5. **测试策略** → 产出测试用例文档，建立 TDD 流程
6. **文档交付** → 产出用户手册、管理员手册
